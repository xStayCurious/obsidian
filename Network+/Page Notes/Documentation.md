DOCUMENTATION
-----------------------------------

Documentation refers to:
- records
- diagrams
- inventories
- operational references

used to:
- manage
- maintain
- troubleshoot
- organize networks

-----------------------------------
CORE PURPOSE
-----------------------------------

Documentation helps:
- reduce downtime
- improve troubleshooting
- standardize operations
- support scalability
- support disaster recovery
- improve communication between teams

-----------------------------------
WHY DOCUMENTATION MATTERS
-----------------------------------

Poor documentation can cause:
- longer outages
- configuration mistakes
- poor troubleshooting
- duplicated work
- security issues
- operational confusion

PHYSICAL DIAGRAMS
-----------------------------------

Physical diagrams show:
- the actual physical layout of network equipment

-----------------------------------
PHYSICAL DIAGRAM PURPOSE
-----------------------------------

Physical diagrams help identify:
- where equipment is located
- how devices are physically connected
- cable routing
- rack placement
- hardware placement

-----------------------------------
WHAT PHYSICAL DIAGRAMS SHOW
-----------------------------------

Examples:
- rack locations
- switch locations
- AP mounting positions
- cable runs
- server room layouts

-----------------------------------
PHYSICAL DIAGRAM EXAM IDEA
-----------------------------------

Physical diagrams answer:
"Where is the equipment physically located?"

-----------------------------------
LOGICAL DIAGRAMS
-----------------------------------

Logical diagrams show:
- how systems communicate logically

-----------------------------------
LOGICAL DIAGRAM PURPOSE
-----------------------------------

Logical diagrams help explain:
- data flow
- VLAN relationships
- subnet structures
- routing paths
- WAN relationships
- logical topology

-----------------------------------
WHAT LOGICAL DIAGRAMS SHOW
-----------------------------------

Examples:
- VLAN segmentation
- subnet assignments
- routing relationships
- firewall zones
- WAN connectivity

-----------------------------------
LOGICAL DIAGRAM EXAM IDEA
-----------------------------------

Logical diagrams answer:
"How does the network logically function?"

-----------------------------------
PHYSICAL VS LOGICAL
-----------------------------------

Physical diagram:
- hardware placement
- cables
- rooms
- racks

Logical diagram:
- communication flow
- VLANs
- subnets
- routing

IPAM
-----------------------------------

IPAM =
IP Address Management

-----------------------------------
IPAM PURPOSE
-----------------------------------

IPAM helps:
- track IP addresses
- manage subnets
- prevent address conflicts
- organize address usage

-----------------------------------
WHY IPAM MATTERS
-----------------------------------

Without IPAM:
- duplicate IPs may occur
- address tracking becomes difficult
- troubleshooting becomes harder

-----------------------------------
WHAT IPAM TRACKS
-----------------------------------

Examples:
- assigned IP addresses
- available addresses
- DHCP scopes
- VLAN/subnet assignments
- DNS relationships

-----------------------------------
IPAM BENEFITS
-----------------------------------

Benefits:
- easier troubleshooting
- organized address management
- improved scalability
- reduced addressing conflicts

COMMON EXAM TRAPS
-----------------------------------

Trap:
Confusing physical and logical diagrams.

Physical:
- hardware layout

Logical:
- communication relationships

-----------------------------------

Trap:
Thinking IPAM only stores DHCP information.

Wrong.

IPAM manages:
- addressing
- subnets
- assignments
- address tracking

-----------------------------------

Trap:
Thinking SLAs are technical configurations.

Wrong.

SLAs define:
- service expectations and obligations

-----------------------------------

Trap:
Thinking wireless surveys only measure signal strength.

Wrong.

Wireless surveys may also analyze:
- interference
- channel usage
- roaming
- AP overlap

-----------------------------------

Trap:
Thinking asset inventory only tracks hardware names.

Wrong.

Asset inventory may include:
- serials
- warranties
- MAC addresses
- software versions
- lifecycle information
