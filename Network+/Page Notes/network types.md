WIRELESS NETWORK TYPES
-----------------------------------

Wireless network types describe:
- how wireless devices communicate
- how wireless infrastructure is organized
- how wireless traffic flows

The guide specifically covers:
- mesh networks
- ad hoc networks
- point-to-point wireless networks

These are extremely exam-relevant because CompTIA frequently tests:
- infrastructure requirements
- redundancy
- scalability
- direct device communication
- deployment scenarios

-----------------------------------
MESH NETWORKS
-----------------------------------

Mesh networks consist of:
- multiple wireless nodes
that:
- dynamically connect to one another

-----------------------------------
CORE PURPOSE OF MESH
-----------------------------------

Mesh networking is designed to provide:
- redundancy
- scalability
- extended wireless coverage
- fault tolerance

-----------------------------------
HOW MESH WORKS
-----------------------------------

In a mesh network:
- nodes communicate with multiple nearby nodes
- traffic may take multiple possible paths
- nodes help relay traffic for other nodes

-----------------------------------
SELF-HEALING
-----------------------------------

The guide emphasizes that mesh networks are:
- self-healing

-----------------------------------
SELF-HEALING MEANING
-----------------------------------

If one mesh node fails:
- traffic may automatically reroute
through:
- another node/path

-----------------------------------
WHY SELF-HEALING MATTERS
-----------------------------------

This improves:
- reliability
- availability
- resilience

-----------------------------------
MESH REDUNDANCY
-----------------------------------

Mesh networks support:
- multiple communication paths

This reduces:
- single points of failure

-----------------------------------
MESH SCALABILITY
-----------------------------------

Mesh networks are:
- highly scalable

New nodes can often be added:
- without major redesign

-----------------------------------
MESH USE CASES
-----------------------------------

The guide lists:
- smart cities
- IoT applications
- large wireless coverage areas

-----------------------------------
AD HOC NETWORKS
-----------------------------------

Ad hoc wireless networks are:
- decentralized wireless networks

-----------------------------------
CORE PURPOSE OF AD HOC
-----------------------------------

Ad hoc networks allow:
- direct device-to-device communication
without:
- centralized infrastructure

-----------------------------------
NO ACCESS POINT REQUIRED
-----------------------------------

The most important exam fact:

Ad hoc wireless networking:
- does NOT require an access point

-----------------------------------
HOW AD HOC WORKS
-----------------------------------

Devices communicate:
- directly with each other

instead of:
- through an AP/router

-----------------------------------
AD HOC CHARACTERISTICS
-----------------------------------

The guide states ad hoc networks:
- are decentralized
- do not rely on existing infrastructure
- allow direct communication between nodes

-----------------------------------
AD HOC USE CASES
-----------------------------------

The guide lists:
- temporary setups
- emergency response
- military operations

-----------------------------------
WHY AD HOC IS USEFUL
-----------------------------------

Ad hoc networking is useful when:
- infrastructure is unavailable
- quick deployment is needed
- temporary communication is required

-----------------------------------
AD HOC ADVANTAGES
-----------------------------------

Advantages:
- rapid deployment
- no AP required
- flexible temporary communication

-----------------------------------
AD HOC DISADVANTAGES
-----------------------------------

Disadvantages:
- poor scalability
- limited management
- weaker centralized security
- limited coverage

-----------------------------------
AD HOC EXAM KEYWORDS
-----------------------------------

Ad hoc exam keywords:
- device-to-device
- peer-to-peer wireless
- no access point
- temporary setup
- decentralized

-----------------------------------
POINT-TO-POINT WIRELESS NETWORKS
-----------------------------------

Point-to-point wireless establishes:
- a direct wireless connection
between:
- two devices/sites

-----------------------------------
CORE PURPOSE OF POINT-TO-POINT
-----------------------------------

Point-to-point wireless is designed for:
- dedicated communication links
- site-to-site connectivity

-----------------------------------
HOW POINT-TO-POINT WORKS
-----------------------------------

Two wireless devices:
- communicate directly
using:
- focused wireless transmission

-----------------------------------
POINT-TO-POINT USE CASES
-----------------------------------

Common examples:
- connecting two buildings
- wireless bridging
- campus links
- long-distance directional wireless

-----------------------------------
DIRECTIONAL ANTENNAS
-----------------------------------

Point-to-point deployments commonly use:
- directional antennas

-----------------------------------
WHY DIRECTIONAL ANTENNAS ARE USED
-----------------------------------

Directional antennas:
- focus signal strength
- improve range
- reduce unwanted interference

-----------------------------------
POINT-TO-POINT ADVANTAGES
-----------------------------------

Advantages:
- long-distance connectivity
- no cabling between sites
- focused communication
- cost-effective site linking

-----------------------------------
POINT-TO-POINT DISADVANTAGES
-----------------------------------

Potential disadvantages:
- alignment requirements
- line-of-sight concerns
- environmental interference

-----------------------------------
NETWORK TYPE COMPARISON
-----------------------------------

Mesh:
- multiple interconnected nodes
- redundant/self-healing

Ad hoc:
- direct device communication
- no AP

Point-to-point:
- direct wireless link between two locations

Infrastructure:
- AP-based centralized wireless networking

-----------------------------------
HIGH-YIELD NETWORK+ FACTS
-----------------------------------

Very important exam facts:

- Mesh = self-healing and redundant
- Ad hoc = no AP required
- Point-to-point = direct wireless bridge
- Infrastructure mode = standard AP-based WiFi
- Directional antennas are common in point-to-point links
- Mesh improves redundancy and coverage
- Ad hoc is useful for temporary deployment

COMMON EXAM TRAPS
-----------------------------------

Trap:
Thinking ad hoc uses access points.

Wrong.

Ad hoc:
- device-to-device directly

-----------------------------------

Trap:
Thinking mesh means only one communication path.

Wrong.

Mesh:
- multiple redundant paths

-----------------------------------

Trap:
Thinking point-to-point means many wireless clients.

Wrong.

Point-to-point:
- direct connection between two endpoints

-----------------------------------

Trap:
Thinking infrastructure mode is decentralized.

Wrong.

Infrastructure:
- centralized AP-based networking

-----------------------------------

Trap:
Thinking mesh always means wired backhaul.

Wrong.

Mesh commonly uses:
- wireless inter-node communication

What is a mesh wireless network?

What does self-healing mean in mesh networking?

Traffic reroutes automatically if a node/path fails.

-----------------------------------

What is an ad hoc wireless network?

A direct device-to-device wireless network without an AP.

An access point.

Q:
What is a common use case for ad hoc networks?

Temporary or emergency communication.

A direct wireless connection between two sites/devices.

Directional antennas.

Wireless networking using access points.

Infrastructure mode.
