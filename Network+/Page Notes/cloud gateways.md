-----------------------------------
CLOUD GATEWAYS
-----------------------------------

Cloud gateways are:
- intermediary devices or services
that:
- connect cloud environments with different networks

-----------------------------------
CORE PURPOSE
-----------------------------------

Cloud gateways facilitate:
- communication
- data transfer
- management

between:
- cloud environments
- private data centers
- other cloud services
- external networks

-----------------------------------
WHAT CLOUD GATEWAYS DO
-----------------------------------

Cloud gateways help:
- users access cloud resources
- applications access cloud resources
- networks communicate securely
- networks communicate efficiently

-----------------------------------
SIMPLE MENTAL MODEL
-----------------------------------

Cloud Gateway =
Bridge between cloud resources and other networks.

-----------------------------------
WHY CLOUD GATEWAYS EXIST
-----------------------------------

Cloud gateways are used to:
- connect cloud and on-premises environments
- support secure communication
- manage cloud traffic
- improve connectivity

-----------------------------------
INTERMEDIARY ROLE
-----------------------------------

The guide describes cloud gateways as:
- intermediary devices or services

Meaning:
- they sit between environments
- helping traffic move between them

-----------------------------------
CONNECTED ENVIRONMENTS
-----------------------------------

Cloud gateways can connect:
- cloud services
- private data centers
- external networks
- other cloud environments

-----------------------------------
COMMUNICATION FUNCTIONS
-----------------------------------

Cloud gateways help ensure:
- secure communication
- efficient communication
- reliable data transfer

-----------------------------------
CLOUD RESOURCE ACCESS
-----------------------------------

Cloud gateways help:
- users access cloud resources
- applications connect to cloud services

-----------------------------------
INTERNET GATEWAY
-----------------------------------

An Internet Gateway:
- connects a VPC to the internet
- enables internet access for cloud resources

-----------------------------------
INTERNET GATEWAY PURPOSE
-----------------------------------

Internet Gateways facilitate:
- communication between cloud instances
and:
- external networks

-----------------------------------
NAT GATEWAY
-----------------------------------

A NAT Gateway:
- allows outbound internet access
- prevents inbound internet connections
- hides private IP addresses

-----------------------------------
NAT GATEWAY PURPOSE
-----------------------------------

NAT Gateways are useful for:
- systems needing updates
- outbound-only internet access
- protecting private instances

-----------------------------------
VPN CONNECTIVITY
-----------------------------------

VPNs create:
- safe
- encrypted connections

between:
- remote users/sites
and:
- private cloud networks

-----------------------------------
PRIVATE-DIRECT CONNECTIONS
-----------------------------------

Private-direct cloud connections:
- bypass the public internet
- improve reliability
- improve security
- improve speed

-----------------------------------
WHY PRIVATE CONNECTIONS MATTER
-----------------------------------

Private-direct connections are useful for:
- businesses with strict security requirements
- performance-sensitive applications
- reliable cloud access

-----------------------------------
CLOUD CONNECTIVITY OPTIONS
-----------------------------------

The guide lists:
- VPN
- Private-direct connections

as major cloud connectivity methods.

-----------------------------------
SECURITY PURPOSE
-----------------------------------

Cloud gateways help:
- secure cloud communications
- protect cloud resources
- manage controlled access

-----------------------------------
NETWORK MANAGEMENT
-----------------------------------

Cloud gateways also support:
- cloud traffic management
- cloud communication control
- cloud connectivity organization

-----------------------------------
COMMON EXAM SCENARIOS
-----------------------------------

Scenario:
"A company wants to securely connect its private data center to cloud resources."

Answer:
Cloud Gateway

-----------------------------------

Scenario:
"A company needs internet access for resources inside a VPC."

Answer:
Internet Gateway

-----------------------------------

Scenario:
"A company wants private instances to access the internet without allowing inbound connections."

Answer:
NAT Gateway

-----------------------------------

Scenario:
"A company needs encrypted communication between remote users and cloud resources."

Answer:
VPN

-----------------------------------

Scenario:
"A company wants a dedicated secure cloud connection that bypasses the internet."

Answer:
Private-direct connection

-----------------------------------
COMMON EXAM TRAPS
-----------------------------------

Trap:
Thinking cloud gateways only connect to the internet.

Wrong.

They can also connect:
- private data centers
- cloud services
- external networks
- other cloud environments

-----------------------------------

Trap:
Confusing Internet Gateway and NAT Gateway.

Internet Gateway:
- enables internet connectivity

NAT Gateway:
- provides outbound-only internet access

-----------------------------------

Trap:
Thinking VPNs are the same as private-direct connections.

Wrong.

VPN:
- encrypted connection over public internet

Private-direct connection:
- dedicated connection bypassing public internet

HIGH-YIELD FLASHCARDS
-----------------------------------

Q:
What is a cloud gateway?

A:
An intermediary device or service connecting cloud environments to other networks.

-----------------------------------

Q:
What is the purpose of a cloud gateway?

A:
To facilitate communication, data transfer, and management between environments.

-----------------------------------

Q:
What does an Internet Gateway do?

A:
Connects a VPC to the internet.

-----------------------------------

Q:
What does a NAT Gateway do?

A:
Provides outbound internet access while hiding private IP addresses.

-----------------------------------

Q:
What is a VPN used for in cloud environments?

A:
Creating encrypted connections between remote users/sites and private networks.

-----------------------------------

Q:
What is a private-direct cloud connection?

A:
A dedicated connection that bypasses the public internet.

-----------------------------------

Q:
Why are cloud gateways important?

A:
They enable secure and efficient cloud connectivity.

-----------------------------------

Q:
What environments can cloud gateways connect?

A:
Cloud services, private data centers, external networks, and other cloud environments.

-----------------------------------

Q:
What is the purpose of a NAT Gateway?

A:
Allow outbound internet access without allowing inbound internet connections.

-----------------------------------

Q:
What is the purpose of an Internet Gateway?

A:
Enable communication between cloud resources and the internet.
